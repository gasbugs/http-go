# Contributing

이 저장소에 관심을 가져주셔서 감사합니다. 세부 기여 지침은 `AGENTS.md`에 정리되어 있으며, 아래는 그 요약과 기본 흐름입니다.

## 빠른 시작
- Go 1.21+와 Docker가 설치되어 있는지 확인합니다.
- `go mod tidy`로 의존성을 정리하고, `go test ./...`로 기본 테스트를 수행합니다.
- 로컬에서 확인할 때는 `go run main.go` 또는 `docker run --rm -p 8080:8080 http-go:dev`를 사용하세요.

## 개발 규칙
- 새 코드 작성 전 `AGENTS.md`의 프로젝트 구조, 스타일, 테스트 지침을 확인합니다.
- 간단한 라우팅은 `main.go`에 두고, 복잡한 로직은 `internal/<feature>/` 패키지로 분리합니다.
- 커밋 전 `go fmt ./...`을 실행하고, 의미 있는 로그만 남깁니다.

## 변경 제출
1. 변경 목적과 테스트 결과를 명확히 남깁니다.
2. 커밋 메시지는 “Add health probe”처럼 짧은 명령형 문장을 사용합니다.
3. PR에는 실행한 명령(`go test ./...`, `docker build …`)과 `curl` 예시를 첨부합니다.
4. 관련 이슈가 있다면 링크로 연결합니다.

## 릴리스 & 배포
- 이미지를 배포하려면 `./build.sh <tag>`를 실행하고, `deployment.yaml`에 새 태그를 반영합니다.
- 배포 후 첫 요청 로그를 확인해 서비스가 정상적으로 응답하는지 검증하세요.

추가 질문이 있으면 이슈를 생성하거나 기존 스레드에 댓글을 남겨 합의한 뒤 작업을 진행해 주세요.
